Changelog — Claude Code Backup System

Breaking [2.0.0] — 2026-03-18

Major reliability overhaul. The backup system is now foolproof — it works without a logged-in user, NAS operations fail fast instead of hanging, email notifications are guaranteed to arrive after backup completion, and NAS retention is based on successful backup count rather than age.

Changed NAS Mounting — Direct mount_smbfs Only

Before

Mounted to /tmp/claude_nas_mount

Fell back to osascript Finder mount when sandbox blocked writes

Finder mount requires GUI session — hung for 1h49m on 2026-03-18 when no user was logged in

After

Mounts to ~/mounts/claude-backup (user-owned, no sudo)

Pre-checks NAS connectivity with nc -z — skips instantly if offline

Wraps mount_smbfs with timeout 30s — no hangs possible

Works identically whether logged in or not

Config KeyOld ValueNew Value
NAS_MOUNT_POINT/tmp/claude_nas_mount(removed)
NAS_MOUNT_BASE(new)$HOME/mounts/claude-backup
NAS_CONNECT_TIMEOUT(new)10 seconds
NAS_MOUNT_TIMEOUT(new)30 seconds

Changed Notification — Inline After Completion

Before

Separate LaunchAgent (com.dt74.claude-backup-notify) ran on a fixed 6:45 AM timer

If backup took longer than 15 minutes, notification found an incomplete log and crashed silently — no email sent

After

Backup script calls notification directly as its final step

Email is guaranteed to contain complete results

Guard checks for ──── END ──── marker before parsing

Accepts log path as $1 — no guessing which log to parse

Removed com.dt74.claude-backup-notify.plist — no longer needed.

Changed NAS Retention — 7 Successful Backups

Before

Pruned NAS backups older than 30 days by file modification time

Unlimited accumulation within 30 days

Failed backups counted the same as successful ones

After

Keeps exactly 7 successful backups (configurable via NAS_RETENTION)

.backup_complete marker written only after successful rsync

Only marked directories count toward retention

Incomplete backups (no marker) cleaned up automatically

NAS_RETENTION(new)7

Added Duration Tracking

Every backup records total duration using the bash $SECONDS built-in. Duration appears in the log summary block (DURATION=0m 44s) and the email notification report.

Added EXIT Trap Cleanup

A comprehensive EXIT trap now handles cleanup on any exit — normal, error, or signal:

Migration Guide

If upgrading from v1.x:

  1. Create the mount directory: 
    mkdir -p ~/mounts/claude-backup
  2. Unload and remove the notify LaunchAgent: 
    launchctl unload ~/Library/LaunchAgents/com.dt74.claude-backup-notify.plist
rm ~/Library/LaunchAgents/com.dt74.claude-backup-notify.plist
  3. Update backup.conf — replace NAS_MOUNT_POINT with: 
    NAS_MOUNT_BASE="$HOME/mounts/claude-backup"
NAS_CONNECT_TIMEOUT=10
NAS_MOUNT_TIMEOUT=30
NAS_RETENTION=7
  4. Copy the new scripts (claude-backup.sh, claude-backup-notify.sh, validate-backup.sh) to _backup_tools/.
  5. Reload the backup LaunchAgent: 
    launchctl unload ~/Library/LaunchAgents/com.dt74.claude-backup.plist
launchctl load ~/Library/LaunchAgents/com.dt74.claude-backup.plist
  6. Run the validation: 
    ~/Projects/claude_backup/_backup_tools/validate-backup.sh
