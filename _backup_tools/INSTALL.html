What Gets Backed Up

Your ~/.claude directory contains everything that makes Claude Code yours:

ContentExamples
SkillsCustom skills you've built or installed
AgentsCustom agent configurations
PluginsInstalled plugins and local plugins
CommandsCustom slash commands
HooksPreToolUse / PostToolUse hooks
Settingssettings.json, settings.local.json
Conversation historyProject contexts, file history, debug logs
Plans & tasksActive plans and task lists

How It Works

~/.claude/ source (your Claude Code config) │ │ rsync (mirrors content) ▼ ~/Projects/claude_backup/ git repo (the backup) │ ├── git commit + push ──────────► GitHub (private repo, versioned history) │ └── rsync over SMB ────────────► NAS (optional secondary backup)
  • Daily at 2:00 AM — backup script runs via macOS LaunchAgent
  • Daily at 7:30 AM — notification script emails a status report
  • Git history provides point-in-time restore capability
  • Each step is independent — Git failure doesn't block NAS, and vice versa

Installation

1 Clone This Repository

git clone https://github.com/your-username/claude-code-backup.git ~/Projects/claude_backup

Or fork it first if you want your own copy, then clone your fork.

2 Configure backup.conf Required

nano ~/Projects/claude_backup/_backup_tools/backup.conf

Update these values to match your environment:

# ── Source & Destination ──────────────────────────────────
BACKUP_SOURCE="$HOME/.claude"
BACKUP_REPO="$HOME/Projects/claude_backup"       # ← where you cloned

# ── NAS Settings ─────────────────────────────────────────
NAS_BACKUP_ENABLED=true                           # ← false if no NAS
NAS_SHARE="//YOUR-NAS-IP/YOUR-SHARE"              # ← your SMB share
NAS_SUBDIR="claude_code_backup"                   # ← subfolder
NAS_KEYCHAIN_ACCOUNT="your-nas-username"         # ← NAS user

# ── Email Notification ───────────────────────────────────
NOTIFY_EMAIL="you@gmail.com"                      # ← your email
GMAIL_KEYCHAIN_ACCOUNT="you@gmail.com"           # ← your email

If you don't have a NAS, just set NAS_BACKUP_ENABLED=false — everything else works without it.

3 Update LaunchAgent Plists Required

The plist files contain hardcoded paths. Update them to match your username:

cd ~/Projects/claude_backup/_backup_tools

# Replace paths in both plists
sed -i '' "s|/Users/youruser|$HOME|g" com.claude-backup.plist
sed -i '' "s|/Users/youruser|$HOME|g" com.claude-backup-notify.plist

Verify the paths look correct:

grep -A1 "ProgramArguments\|StandardOutPath" com.claude-backup.plist

4 Set Up Your GitHub Remote

If you forked the repo, the remote is already set. Otherwise:

cd ~/Projects/claude_backup

# Create your own private repo
gh repo create YOUR-USERNAME/claude-code-backup --private

# Update the remote
git remote set-url origin https://github.com/YOUR-USERNAME/claude-code-backup.git

5 Create a Gmail App Password Optional

Required only if you want email notifications. Gmail needs an App Password when 2FA is enabled:

  1. Go to myaccount.google.com/apppasswords
  2. Select app → Other → enter "claude-backup"
  3. Click Generate
  4. Copy the 16-character password — the setup script will prompt for this

6 Run the Interactive Setup

bash ~/Projects/claude_backup/_backup_tools/setup.sh

The setup script will:

  1. Install msmtp via Homebrew
  2. Initialize the git repo and verify the remote
  3. Prompt for your NAS password → stores in macOS Keychain
  4. Prompt for your Gmail App Password → stores in Keychain
  5. Configure msmtp for email notifications
  6. Create the log directory
  7. Symlink and load the LaunchAgent schedules
  8. Run an initial test backup

Usage

Automatic (Default)

  • 2:00 AM daily — full backup (rsync → git push → NAS sync)
  • 7:30 AM daily — email notification with backup status

Manual / Ad-Hoc

# Full backup
~/Projects/claude_backup/_backup_tools/claude-backup.sh

# GitHub only (skip NAS)
NAS_BACKUP_ENABLED=false ~/Projects/claude_backup/_backup_tools/claude-backup.sh

# Skip conversation cache (smaller, faster)
INCLUDE_CONVERSATION_CACHE=false ~/Projects/claude_backup/_backup_tools/claude-backup.sh

# Send notification email now
~/Projects/claude_backup/_backup_tools/claude-backup-notify.sh

Security

  • No plaintext passwords — NAS and Gmail credentials stored in macOS Keychain, retrieved at runtime
  • SMB passwords URL-encoded — special characters handled correctly (a common macOS gotcha)
  • Credentials cleared from memory — variables unset immediately after use
  • Dedicated msmtp config — stored in _backup_tools/.msmtprc (gitignored), won't conflict with system config
  • Lockfile protection — prevents concurrent backup runs from corrupting state
  • Pre-commit size check — files over 90 MB flagged before committing (GitHub limit is 100 MB)

Customization

Change the Backup Schedule

Edit the hour/minute in the plist, then reload:

# Edit the plist
nano ~/Projects/claude_backup/_backup_tools/com.claude-backup.plist

# Reload (required for plist changes to take effect)
launchctl unload ~/Library/LaunchAgents/com.claude-backup.plist
launchctl load ~/Library/LaunchAgents/com.claude-backup.plist

Changes to bash scripts take effect immediately. Only plist changes need a reload.

Disable NAS Backup

Set in backup.conf:

NAS_BACKUP_ENABLED=false

Or override per-run:

NAS_BACKUP_ENABLED=false ~/Projects/claude_backup/_backup_tools/claude-backup.sh

Use a Different Email Provider

Edit _backup_tools/.msmtprc after running setup:

account        outlook
host           smtp-mail.outlook.com
port           587
from           you@outlook.com
user           you@outlook.com
passwordeval   security find-generic-password -s claude-backup-email -a you@outlook.com -w

account default : outlook

Troubleshooting

IssueCauseFix
NAS mount "No route to host" but server pings Password has special characters Script URL-encodes automatically; verify password in Keychain is correct
NAS mount "Permission denied" on mkdir Mount point under /Volumes/ Use /tmp/ mount point (default in config)
NAS subfolder access denied Service account lacks permissions Fix share ACLs on the NAS for the service account
Gmail auth fails Not using App Password Generate at myaccount.google.com/apppasswords
Schedule change not working Plist not reloaded launchctl unload then launchctl load
git / msmtp not found at 2 AM PATH not set in LaunchAgent Verify EnvironmentVariables in plist includes /opt/homebrew/bin
Backup script hangs Stale lockfile from a crash Delete ~/Library/Logs/claude-backup/.claude-backup.lock

Viewing Logs

# Latest backup log
ls -t ~/Library/Logs/claude-backup/backup_*.log | head -1 | xargs cat

# All available logs (last 30 days retained)
ls ~/Library/Logs/claude-backup/
